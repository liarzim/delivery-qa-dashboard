# לוח בקרה QA ודליברי

מערכת לוח בקרה מתקדמת למעקב אחר מדדי QA ודליברי, עם בונה ווידג'טים מותאם אישית.

---

## דרישות מוקדמות

- [Node.js](https://nodejs.org/) גרסה 20 ומעלה
- חיבור לרשת מקומית (לגישה לקבצי Excel)

---

## התקנה ראשונה

פתח טרמינל בתיקיית הפרויקט והרץ:

```bash
npm run install:all
```

פקודה זו מתקינה את כל התלויות בתיקיות `root`, `server` ו-`client`.

לאחר מכן, בנה את הממשק:

```bash
npm run build
```

---

## הפעלה

### הפעלה שקטה (ללא חלון טרמינל)

לחץ פעמיים על הקובץ:

```
start.vbs
```

הדפדפן ייפתח אוטומטית בכתובת `http://localhost:3001`.

### הפעלה במצב פיתוח (שני חלונות)

```bash
npm run dev
```

פקודה זו מפעילה גם את שרת ה-Express (פורט 3001) וגם את שרת Vite (פורט 5173) במקביל.

---

## הגדרת קבצי Excel

1. כנס ל-**הגדרות** (זמין למנהל מערכת בלבד)
2. בקטגוריה **מקורות נתונים**, הגדר את שמות הקבצים:
   - **קובץ דליברי** — חוברת Excel עם גיליונות `FLOW` ו-`Commitment Summary`
   - **קובץ באגים QA** — חוברת Excel עם גיליון `Bug Data`
   - **קובץ Escaping QA** — חוברת Excel עם נתוני Escaping Defects
3. לחץ **שמור את כל ההגדרות**
4. לחץ **טען נתונים מחדש** בסרגל הצד

---

## בונה ווידג'טים מותאם אישית

הכלי מאפשר ליצור גרפים ומדדים מותאמים אישית על בסיס הנתונים הגולמיים.

### יצירת ווידג'ט חדש

1. לחץ על **בונה ווידג'טים** בסרגל הצד
2. בחר **מקור נתונים** (דליברי / באגים QA / Escaping Defects)
3. בחר **סוג גרף** (עמודות / קו / שטח / עוגה / KPI / טבלה)
4. הגדר את שדות **ציר X** (קיבוץ) ו**ציר Y** (מדד)
5. בחר **נוסחת אגרגציה** (ספירה / סכום / ממוצע / מינימום / מקסימום)
6. השתמש בגריד הנתונים בחצי התחתון לסינון ממוקד
7. לחץ **שמור** לשמירת הווידג'ט האישי

### פרסום לבנק הגלובלי

לאחר שמירה, לחץ **פרסם לבנק הגלובלי** — הווידג'ט ישלח לאישור מנהל המערכת.

### אישור ווידג'טים (מנהל מערכת)

1. כנס ל-**הגדרות ← אישור ווידג'טים**
2. אשר או דחה ווידג'טים ממתינים
3. ווידג'טים מאושרים יופיעו בבנק הווידג'טים של כל המשתמשים
